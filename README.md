# 会议纪要 & 项目进度闭环 Agent
基于OpenClaw+LanceDB的个人Markdown知识管理与记忆增强Agent。它解决了个人笔记散落在本地、跨时间跨主题信息无法关联、知识沉淀与复盘效率低的痛点。核心逻辑流为单Agent长链推理工作流：每日自动同步本地Markdown文件并完成分块向量化存储；支持自然语言提问，Agent通过长链推理召回相关笔记并整合上下文生成结构化回答；每周自动生成复盘报告，梳理关键事件并关联过往笔记辅助规划。目前已接入个人近3年的1200+篇笔记，日常信息检索效率提升90%，每周复盘时间从1小时缩短至15分钟，月均消耗约30万Token。

团队会议纪要与项目进度闭环 Agent 项目说明
一、项目解决核心痛点
传统线下 / 线上会议无自动纪要能力，人工整理会议内容耗时久、效率低，单次会议整理常需半小时以上；
会议中的待办任务、责任人、截止时间依赖人工记录，易遗漏、易遗忘，没有统一归档；
项目任务分散在聊天记录、文档中，缺少统一存储与历史回溯能力，无法沉淀项目会议资产；
缺少任务进度风险预判，临近截止的延期隐患不能提前发现，导致项目进度被动延期；
会议待办任务无法自动同步到团队协作工具，需手动二次录入，重复工作量大。
二、核心逻辑
采用多 Agent 分工协作架构
拆分五大独立 Agent 各司其职、串行闭环调度：
会议转录 Agent：抓取并整理飞书会议完整文本；
信息抽取 Agent：基于大模型长链推理做语义理解，从杂乱会议话术里精准提取待办任务、责任人、截止时间、依赖关系；
LanceDB 存储 Agent：将会议原文 + 结构化任务向量化存入向量数据库，支持历史检索与上下文关联；
飞书任务同步 Agent：自动把结构化任务同步至飞书多维表格并推送群提醒；
风险预警 Agent：基于向量库历史数据做语义检索，智能识别临近截止、存在延期风险的任务并预警。
关键技术特性
内置长链推理：突破简单关键词匹配，能理解口语化会议表述，精准结构化抽取信息；
多 Agent 流水线闭环：转录→抽取→存储→同步→预警全自动化流转，无需人工介入；
向量数据库加持：实现会议知识沉淀、跨会议信息关联、智能风险检索；
深度对接飞书生态，贴合团队日常办公场景，落地性强。
三、项目技术栈
飞书开放平台 + Python + OpenAI 兼容大模型 + LanceDB 向量数据库 + 多 Agent 编排架构


功能说明：
1. 飞书会议内容自动转录
2. 大模型抽取任务、责任人、截止时间
3. 存入LanceDB向量库
4. 同步到飞书多维表格
5. 自动任务延期风险预警
